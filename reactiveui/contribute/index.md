# Contributing to ReactiveUI

ReactiveUI is a collection of around 15 actively maintained projects under the [reactiveui](https://github.com/reactiveui) GitHub organisation. We welcome contributions of all kinds — bug fixes, features, documentation, and test coverage.

## Quick start

1. Find something to work on — check [issues labelled `help wanted`](https://github.com/search?q=org%3Areactiveui+label%3A%22help+wanted%22+state%3Aopen&type=issues) across the org
2. **For anything beyond a trivial fix, open an issue or discussion first** — see [pull requests](pull-requests.md) for why this matters
3. Read the [building & testing](building-and-testing.md) guide to get a repo compiling locally
4. Follow the [code style](software-style-guide/code-style.md) and [commit conventions](software-style-guide/commit-message-convention.md)
5. Open a [pull request](pull-requests.md)

## Key repositories

|---|---|
| [ReactiveUI](https://github.com/reactiveui/ReactiveUI) | Core framework — ReactiveObject, ReactiveCommand, bindings |

| [DynamicData](https://github.com/reactivemarbles/DynamicData) | Reactive collections |
| [ReactiveUI.Validation](https://github.com/reactiveui/ReactiveUI.Validation) | Validation helpers |

| [Sextant](https://github.com/reactiveui/Sextant) | View-model first navigation |

## Where to ask questions

Use [GitHub Discussions](https://github.com/reactiveui/ReactiveUI/discussions). Post in the discussion section of the specific repo your question relates to. For general questions, use the [main ReactiveUI discussions](https://github.com/reactiveui/ReactiveUI/discussions).

## Code of Conduct

All participants are expected to follow our [Code of Conduct](../Code-Of-Conduct). If you experience or witness unacceptable behaviour, contact us at [hello@reactiveui.net](mailto:hello@reactiveui.net).
